# 步骤 3：AIS 转换与步骤 4：验收标准参考

> 由 SKILL.md 步骤 3/4 触发加载。提供 AIS 转换示例和业务验收标准写作参考。

## AIS 转换示例

| 原始诉求 | → | As a | I want | So that |
|---------|---|------|--------|---------|
| 每天手动复制 5 个系统的数据到 Excel 对比 | 负责多项目运营的经理 | 把多个项目的关键数据整理成一个可对比的清单 | 周会上不用逐个截图，一眼看到差异 |
| 反复追问审批人进度 | 提交申请的员工 | 提交后能实时看到审批到哪一步了 | 不用发消息问、也不用担心被遗忘 |
| 线下签字审批流程慢 | 部门主管 | 在提交审批后自动通知到对应审批人 | 不用挨个打电话催、审批不卡在人找不到上 |

## GWT 格式示例

```
Given 订单已签收且在 7 天内
When 商户点击"申请退款"
Then 系统在 24 小时内完成审批并通知商户结果
```

## 自然语言格式示例（业务规则不明确时）

```
- 申请提交后，对应的审批人必须在 24 小时内收到通知
- 审批结果（通过/拒绝）必须以消息形式推送给申请人
- 超过 48 小时未审批的申请，系统需要提醒审批人
```
