# 恢复场景

> SKILL.md 回退路径时加载。文件损坏或同步失败后的详细恢复方案。

## 文件损坏恢复

### 场景 1：合并后文件不可读
```
1. 定位备份文件：`ls {filename}.bak.*`（按时间排序取最新）
2. 恢复：`cp {filename}.bak.{timestamp} {filename}`
3. 检查 MANIFEST.json 版本号是否已更新
4. 若版本号已更新，回退到之前的版本号
```

### 场景 2：冲突标记未解决
```
1. 搜索未解决的冲突标记：`grep -rn "<<<<<<<" docs/`
2. 按上下文判断保留哪个版本
3. 删除冲突标记后保存
4. 通知提交者手动确认
```

### 场景 3：MANIFEST.json 损坏
```
1. 从版本控制恢复：`git checkout -- MANIFEST.json`
2. 或从备份文件恢复
3. 重新执行同步
```

## 重试规则
- 所有回退路径最多重试 3 轮
- 3 轮后仍未解决 → 升级人类处理
- 每轮重试前记录失败原因
