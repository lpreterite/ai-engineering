# 上下文捕获模板

> SKILL.md 步骤 1 时加载。记录卡点涉及的完整上下文信息。

## 必填字段

| 字段 | 说明 |
|------|------|
| 上游仓库 | `lpreterite/ai-engineering`（固定） |
| 下游项目 | 项目名称和版本（如有） |
| 涉及 Skill | `skills/<name>/SKILL.md` + 版本号 |
| 涉及 Agent | 哪个 Agent 角色触发的 |
| 涉及标准文件 | 哪个 guide/*.md 或 role prompt 出现偏差 |
| 矫正次数 | 3+ 次 |
| AI 模型 | 当前使用的模型标识（如 `deepseek-v4-flash`、`claude-sonnet-4`） |
| 人类反馈原文 | 最近的 1-2 次人类矫正的原话 |

## 矫正类型判断

上报条件：同一类目下 ≥ 3 次

| 类目 | 关键词信号 |
|------|-----------|
| Gate 推进 | "还没到 Gate"、"跳步了"、"不能跳过验收" |
| Issue 生命周期 | "不能直接 close"、"状态不对"、"要先 triage" |
| 角色分工 | "应该交给 Tester"、"这不是我该做的" |
| 文档规范 | "不应该写在 Issue 里"、"要更新 PRD"、"版本号不对" |
| 移交协议 | "还没 ready-for-test"、"不能改代码了" |
| 通道隔离 | "不要在评论里改代码"、"评论通道只读" |

## 不报送的信号（实现类）

- 代码实现不符预期
- 技术选型分歧
- UI 样式调整
- 业务逻辑理解偏差
- 工具用法偏好
