# design-spec-templates — 设计规范模板

Type: 2 (Data & Information Management)

## 工作流

### 步骤 0：前置确认 [自由度：低]
- 确认 PRD 已通过 Gate 1 验收
- 确认项目设计系统已定义

### 步骤 1：确认范围 [自由度：中]
- 确认设计覆盖的页面列表
- 确认目标平台（Web/iOS/Android）

### 步骤 2：选择模板 [自由度：低]
- 按场景选择设计规范结构
- → 此时加载 [references/design-spec-example.md](references/design-spec-example.md)

### 步骤 3：填充内容 [自由度：高]
- 设计稿覆盖所有页面和状态
- 标注规范完整
- 交互说明清晰
- → 此时加载 [references/interaction-guide.md](references/interaction-guide.md)

## 模板结构

```
# {项目名} 设计规范 v{版本}

## 1. 设计系统引用
- 颜色系统
- 字体排版
- 间距网格
- 组件库引用

## 2. 页面清单
- 各页面设计稿引用
- 页面状态覆盖（默认/悬停/点击/禁用/空态/加载/错误）

## 3. 交互说明
- 过渡动画
- 微交互
- 手势操作

## 4. 响应式适配
- 断点定义
- 各断点布局变化

## 5. 设计标注
- 间距、颜色、字体标注
- 图标资源引用
```

## 回退路径
- 设计稿不完整 → 标记待补充项，通知 UI/UX 补全（最多重试 2 轮）
- 模板部分章节不适用 → 标记为"不适用"后跳过

## 前置条件
- PRD 已通过 Gate 1 验收
- 设计系统已定义

## 完成标准
- 设计稿覆盖所有页面和关键状态
- 标注规范完整可交付开发
- 交互说明清晰无歧义
