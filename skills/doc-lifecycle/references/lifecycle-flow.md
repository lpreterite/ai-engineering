# 文档生命周期流程

> SKILL.md 步骤 2 时加载。文档从创建到归档的完整阶段说明。

## 生命周期阶段

| 阶段 | 说明 | 负责人 |
|------|------|--------|
| **创建** | 按模板创建文档，填写初始内容 | 相关 Agent 或人类 |
| **评审** | 提交给干系人评审，确认文档质量 | Orchestrator 协调 |
| **发布** | 评审通过后正式发布到文档体系 | Orchestrator |
| **维护** | 根据项目进展更新文档内容 | 相关 Agent 或人类 |
| **归档** | 项目结束后归档供后续参考 | Orchestrator |

## 评审流程

```
创建文档 → 提交评审 → 评审通过 → 发布文档
                │
                ▼
           评审不通过
                │
                ▼
           返回修改
```

## Gate 评审参与方

| Gate | 评审内容 | 参与方 |
|------|----------|--------|
| Gate 1 | PRD | Product Owner, PO Agent |
| Gate 2 | 技术方案 | PM, Developer Agent |
| Gate 2 | 用户方案 | Product Owner, UI/UX Agent |
| Gate 3 | 可使用软件验收 | Product Owner, Orchestrator |
| Gate 4 | 可维护软件验收 | Product Owner, Orchestrator |

## 评审要点

| 评审项 | 说明 |
|--------|------|
| 完整性 | 文档内容是否完整覆盖要求 |
| 准确性 | 内容是否准确无误 |
| 一致性 | 与其他文档是否一致 |
| 可执行性 | 描述是否可操作执行 |

## 同步触发词

AI 检测到以下关键词时，立即检查并更新相关文档：

| 关键词 | 应检查更新的文档 |
|--------|-----------------|
| "完成"、"实现了"、"解决了" | STATUS.md |
| "开始 M2"、"进入下一阶段" | STATUS.md, milestones.md |
| "发现阻塞"、"阻塞项" | STATUS.md |
| "提交"、"commit" | tasklist.md |

## 同步执行流程

1. 检测到项目变更（git commit/功能完成/里程碑达成）
2. 判断变更类型
3. 确定需要更新的文档列表
4. 执行文档更新
5. 记录变更日志
