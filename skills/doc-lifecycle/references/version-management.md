# 版本管理

> SKILL.md 步骤 4 时加载。文档版本格式和递增规则。

## 版本格式

| 格式 | 说明 | 示例 |
|------|------|------|
| v0.x | 草稿阶段 | v0.1, v0.2 |
| v1.x | 正式发布 | v1.0, v1.1 |
| v2.x | 重大修订 | v2.0 |

## 递增规则

- 草稿阶段：v0.1 → v0.2 → v0.3（小幅修改）
- 正式发布：v1.0（首次发布）
- 后续更新：v1.1（小幅修改）、v2.0（重大变更）

## 增量版本原则

每个版本的交付物是独立的增量文档，新版本不修改旧版本：

```
docs/
├── product/
│   ├── PRD-v1.0.md        ← v1.0
│   ├── PRD-v1.1.md        ← v1.1 增量（独立文件）
│   └── PRD-v2.0.md        ← v2.0 大版本
├── engineering/
│   ├── ARCHITECTURE-v1.0.md
│   └── ARCHITECTURE-v1.1.md
```

## 变更记录

每次更新文档时，在末尾修订记录中记录：

```markdown
## 修订记录

| 版本 | 日期 | 修订内容 | 修订人 |
|------|------|----------|--------|
| v0.2 | 2026-04-04 | 新增 xxx 内容 | Orchestrator |
| v0.1 | 2026-04-03 | 初始版本 | PO Agent |
```

## 更新触发条件

| 触发条件 | 说明 | 责任方 |
|----------|------|--------|
| 里程碑完成 | 阶段转换时更新状态 | Orchestrator |
| 任务状态变更 | 任务完成/开始时更新 | Orchestrator |
| 风险状态变更 | 风险识别/解决/升级时 | Orchestrator |
| 人工决策 | 重大决策做出后 | 决策者 |
