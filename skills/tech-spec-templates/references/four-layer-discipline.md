# 4 层纪律

> SKILL.md 完成标准时加载。技术方案必须遵守的四层质量标准。

## 前端纪律

- 组件化设计，避免重复代码
- 状态管理清晰，数据流向单一
- 响应式设计覆盖所有断点
- 可访问性（a11y）达标
- 性能预算设定（加载时间 < 3s，LCP < 2.5s）

## 后端纪律

- API 设计符合 RESTful 或 GraphQL 最佳实践
- 请求验证和错误处理完备
- 日志记录分级（INFO/WARN/ERROR）
- 鉴权与授权覆盖所有敏感操作
- 数据库查询优化（N+1 问题预防）

## 数据库纪律

- 索引策略基于查询模式设计
- 数据迁移方案完备
- 备份与恢复策略明确
- 敏感数据加密存储
- 连接池配置合理

## DevOps 纪律

- CI/CD 流水线配置完整
- 环境变量管理（不硬编码密钥）
- 日志与监控方案到位
- 容器化部署配置
- 回滚方案就绪
