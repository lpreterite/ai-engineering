# tech-spec-templates — 技术方案模板

Type: 2 (Data & Information Management)

## 工作流

### 步骤 1：确认范围 [自由度：中]
- 确认技术方案覆盖的功能范围和约束条件
- 确认目标读者（技术团队/架构评审/PM）

### 步骤 2：选择模板 [自由度：低]
- 按场景选择对应模板
- → 此时加载 [references/tech-spec-example.md](references/tech-spec-example.md)

### 步骤 3：填充内容 [自由度：高]
- 按模板结构逐节填充
- 每个技术决策附带依据

## 模板结构

```
# {项目名} 技术方案 v{版本}

## 1. 概述
- 需求背景
- 目标与范围

## 2. 系统架构
- 架构图（ASCII/文字描述）
- 模块划分与职责

## 3. 技术选型
- 各选项对比表
- 选型依据

## 4. 接口定义
- API 端点
- 数据格式

## 5. 数据模型
- 核心实体
- 关系定义

## 6. 错误处理
- 异常场景
- 降级策略

## 7. 部署方案
- 环境要求
- 部署步骤
```

## 回退路径
- 模板结构与项目需求不匹配 → 切回步骤 1 重新确认范围（最多重试 2 轮）
- 模板中部分章节不适用 → 标记为"不适用"后跳过

## 前置条件
- PRD 已通过 Gate 1 验收
- 架构约束已明确

## 完成标准
- 模板中所有必填章节已填写
- 技术选型附带依据
- → 此时加载 [references/four-layer-discipline.md](references/four-layer-discipline.md)
