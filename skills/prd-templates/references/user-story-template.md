# 用户故事与验收标准指南

> SKILL.md 步骤 3 时加载。用户故事编写和验收标准定义的最佳实践。

## 用户故事编写原则

- 从用户视角描述，而非技术视角
- 清晰、可理解
- 包含验收标准
- 优先级划分合理

## 验收标准编写原则

- 可测试：有明确的通过/不通过判定
- 无歧义：不同人读理解一致
- 包含正向和负向场景
- 边界条件有明确说明
- 性能要求有量化指标

## 验收标准示例

```markdown
### 验收标准：用户登录

正向场景：
- [ ] 输入正确的用户名和密码，点击登录，成功跳转到首页
- [ ] 登录成功后，右上角显示用户头像和昵称

负向场景：
- [ ] 输入错误的密码，显示"用户名或密码错误"
- [ ] 连续 5 次输入错误，账户被锁定 30 分钟

边界条件：
- [ ] 用户名为空时，登录按钮不可用
- [ ] 密码长度超过 128 字符，给出提示
```

## RICE 优先级评分

| 维度 | 说明 | 评分 (1-10) |
|------|------|-------------|
| Reach（覆盖面） | 影响多少用户 | |
| Impact（影响力） | 对单个用户的价值 | |
| Confidence（信心） | 对效果的确信度 | |
| Effort（工作量） | 实现成本（反向评分） | |

总分 = (Reach × Impact × Confidence) / Effort
