# 回滚机制

> SKILL.md 步骤 4 时加载。更新失败后的回滚操作。

## 自动备份

每次更新操作前自动创建备份：

```bash
cp checklists.md checklists.md.bak.20260528
```

备份保存在同一目录，命名格式：`{文件名}.bak.{YYYYMMDD}`

## 回滚触发条件

| 条件 | 说明 |
|------|------|
| 合并后文件损坏 | 文件不可读或格式损坏 |
| 冲突无法解决 | 三路合并后仍有未解决的冲突标记 |
| 下游定制丢失 | 更新后发现定制内容被覆盖 |
| 版本不匹配 | 更新后的版本号与预期不一致 |

## 回滚流程

1. 确认回滚文件和对应的 `.bak.{timestamp}` 备份
2. 用备份文件覆盖更新后的文件
3. 恢复 MANIFEST.json 中的版本号到更新前
4. 记录回滚原因到变更日志

## 验证

回滚后检查：
```
□ 文件内容与备份一致
□ MANIFEST.json 版本号已恢复
□ 项目功能正常
□ 回滚原因已记录
```
