# 三路合并

> SKILL.md 步骤 2 时加载。diff3 三路合并细则。

## 三路合并原理

三路合并（diff3）基于三个版本进行合并：
- **base**：上一个共同版本
- **ours**：当前文件（可能含下游定制）
- **theirs**：上游新版本

```
base (v0.2)       ours (v0.2+custom)       theirs (v0.3)
    │                    │                       │
    └────────┬───────────┘───────────────────────┘
             │
      三路合并器
             │
             ▼
    result (合并结果，可能含冲突标记)
```

## 冲突标记格式

```
<<<<<<< ours
下游定制内容
=======
上游新版本内容
>>>>>>> theirs
```

## 策略选择

| 上游变更 | 下游定制 | 策略 |
|----------|----------|------|
| 无 | 无 | 跳过 |
| 有 | 无 | 直接覆盖 |
| 无 | 有 | 跳过（保留定制） |
| 有 | 有（无重叠） | 自动合并 |
| 有 | 有（有重叠） | 三路合并 → 可能冲突 |

## 自动合并条件

- 上游变更和下游定制在文件的不同区域（行范围不重叠）
- 或重叠区域的内容完全一致

## 冲突处理

- 产生冲突标记后，AI 自动尝试裁决
- 如果能识别哪个版本应保留，直接解决
- 不能识别则保留冲突标记，通知人工处理
