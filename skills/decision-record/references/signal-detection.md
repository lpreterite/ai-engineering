# 决策信号识别

> SKILL.md 步骤 0 时加载。完整信号识别规则。

## 信号类型

| 信号 | 示例 |
|------|------|
| 人类表达明确选择倾向 | "选 X"、"用 Y"、"倾向于方案 A"、"我倾向用 PostgreSQL" |
| 讨论多个方案并排除其一 | "B 不行，成本太高"、"方案 C 不靠谱" |
| 确认了技术选型或设计方向 | "那就用 PostgreSQL"、"就用方案 A 吧" |
| 需要外部确认的疑问 | "这个问题需要确认一下"、"这个得问问 XXX" |
| 比较性讨论 | "A 比 B 好在哪"、"A 和 B 的区别是什么" |
| 约束性讨论 | "这个受限于 XXX"、"在 XXX 前提下" |

## 非决策信号（不触发）

- 纯信息询问（"这个 API 怎么用？"）
- 实现细节讨论（"这个函数参数是什么？"）
- 已有明确答案的确认（"配置文件的路径是这里吧？"）

## 3 轮门槛说明

"有效讨论"指针对同一问题/方案的连续对话轮次：
- 不包含信息收集轮次
- 不包含纯确认轮次
- 3 轮后 AI 主动询问，不强创建

## 创建后的对话

人类确认创建后，不要立即关闭 Issue，告知 Issue 编号后继续讨论。流程保持「过程与结果分离」：
- Issue 记录讨论过程
- 关键文档记录最终结论
