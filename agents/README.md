## 1. 角色矩阵

| 角色 | 人类/Agent | 核心职责 | 主要参与阶段 |
|------|-----------|----------|-------------|
| **Product Owner (PO)** | 人类 | 需求负责、验收把关 | 打磨阶段 |
| **Product Manager (PM)** | 人类 | 流程协调、进度管控 | 全流程 |
| **PO Agent** | Agent | 需求分析、PRD 起草 | 打磨阶段 |
| **Orchestrator Agent** | Agent | 多智能体编排、任务路由、门控管理 | 全流程 |
| **UI/UX Agent** | Agent | 用户方案设计 | 打磨阶段 |
| **全栈开发工程师** | Agent | 全栈技术实施（前端/后端/数据库/DevOps） | 执行阶段 |
| **Tester Agent** | Agent | 测试执行 | 执行阶段 |

## 4. 协作协议

| 协议 | 说明 |
|------|------|
| **消息格式** | 统一使用 JSON，字段见各角色接口定义 |
| **响应时效** | 紧急（阻塞）< 5min，建议 < 1h，状态更新 < 24h |
| **确认机制** | 重要消息需接收方确认（`ack: true`） |
| **上下文传递** | 跨 Agent 共享项目上下文，通过知识库而非消息链 |
| **冲突解决** | 优先基于已批准的文档，其次基于最后更新，争议升级人工 PM（Product Manager） |
